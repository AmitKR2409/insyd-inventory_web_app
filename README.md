# Insyd Inventory Transparency Web App

A full-stack inventory management and transparency dashboard built as part of the **Insyd Assignment**.  
The application focuses on improving **trust, accountability, and decision-making** in inventory data by tracking stock changes with reasons and generating actionable insights.

---

## Problem Statement

### Problem 1  
**Lack of trust in inventory data due to missing transparency and accountability**

### Problem 2  
**Decision paralysis due to lack of actionable insights**

---

## Solution Overview

This application provides:
- Transparent inventory tracking
- Reason-based stock updates
- Complete stock change history
- Automated insights for better decisions

Both **backend APIs** and **frontend dashboard** are fully deployed and connected.

---

## Features

### Inventory Management (Problem 1)
- View inventory items with real-time quantity
- Update stock with positive or negative changes
- Mandatory reason for every stock update
- Automatic logging of stock change history with timestamps

### Actionable Insights (Problem 2)
- Low stock item detection
- Fast-moving items based on change frequency
- Slow-moving items
- Most common stock change reasons (damage, sale, etc.)

## How It Works

1. Inventory items are stored in MongoDB
2. Any stock update requires a **reason**
3. Each update is logged with:
   - Quantity change
   - Reason
   - Timestamp
4. Insights are generated by analyzing stock change history
5. Frontend consumes backend APIs to display data in real time
